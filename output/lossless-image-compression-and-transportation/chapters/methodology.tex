\subsection{System Architecture}

A modular and extensible framework is proposed by this system. The implementation, testing, and direct comparison of lossless image compression algorithms are entirely facilitated by this framework. The overall system architecture is illustrated clearly in Figure~\ref{fig:system_architecture}. Four main layers are contained within this architecture. These include the data layer, the algorithm layer, the evaluation layer, and the visualization layer.

\subsection{Data Layer}

Images are loaded directly from the disk by the data layer. Conversion to a standard format, such as RGB or grayscale, is subsequently performed. A uniform dimension of typically $256 \times 256$ pixels is achieved by resizing the images. Consistent evaluation is completely ensured by this strict standardization process. Pixel values are also normalized to a tight range of $[0,1]$ by this specific layer. This is done strictly for algorithms that demand floatingpoint inputs, such as neural networks.

\subsection{Algorithm Layer}

The full implementations of all lossless compression algorithms are contained entirely within this layer. Every single algorithm is heavily encapsulated inside its own separate class. A strictly consistent interface is utilized across all of them:





\begin{verbatim}
class CompressionAlgorithm:
    def compress(self, image_array):
        # Returns compressed representation and metadata
        pass
    
    def decompress(self, compressed_data, metadata):
        # Returns reconstructed image array
        pass
\end{verbatim}

This design allows the evaluation layer to treat all algorithms uniformly, enabling fair comparisons.

\subsection{Evaluation Layer}

The evaluation layer computes standard quality metrics for each compression result:

\begin{itemize}
    \item Compression ratio: \( \displaystyle R = \frac{\text{original size}}{\text{compressed size}} \)
    
    \item Mean squared error (MSE): \( \displaystyle \text{MSE} = \frac{1}{N}\sum_{i=1}^{N} (I_{ij} - \hat{I}_{ij})^2 \)
    
    \item Peak signal-to-noise ratio (PSNR): \( \displaystyle \text{PSNR} = 20\log_{10}\left(\frac{255}{\sqrt{\text{MSE}}}\right) \)
    
    \item Execution time and throughput (pixels per second)
\end{itemize}

\subsection{Visualization Layer}

The visualization layer generates a comprehensive set of plots, including side-by-side comparisons of original and compressed images, error maps, histograms of pixel values and residuals, convergence curves (for iterative algorithms), and color palettes (for clustering methods). All visualizations are produced using Matplotlib and are saved as high-resolution images for inclusion in the report.

\subsection{Design Approach}

The system follows a \textbf{modular, object-oriented design} to facilitate the addition of new algorithms and the reuse of common components. Key design patterns employed include:

\begin{itemize}
    \item \textbf{Strategy pattern}: Each compression algorithm is a strategy that can be swapped at runtime.
    \item \textbf{Template method pattern}: The evaluation procedure (load, compress, decompress, measure) is a fixed template; only the algorithm varies.
    \item \textbf{Factory pattern}: A factory function creates algorithm instances based on a configuration string (e.g., \texttt{"huffman"}, \texttt{"calic"}).
\end{itemize}

All algorithms are implemented in pure Python using NumPy for numerical operations. For computationally intensive methods (Gath-Geva, PSO, FBPNN), GPU acceleration is provided via JAX (with Metal backend for Apple M4) and PyTorch MPS, with automatic fallback to CPU when GPU is unavailable.

\subsection{Backend Design}

The backend consists of Python modules organized as follows:

\begin{itemize}
    \item \texttt{data/} – Image loading, resizing, normalization, and augmentation.
    
    \item \texttt{algorithms/} – Subdirectories for each family:
    \begin{itemize}
        \item \texttt{entropy/} – Huffman, Arithmetic, LZW.
        \item \texttt{spatial/} – RLE, variable block size, lossy+residual.
        \item \texttt{predictive/} – JPEG lossless (7 predictors).
        \item \texttt{context/} – CALIC, LOCO-I.
        \item \texttt{transform/} – S-Transform, S+P, lifting wavelet.
        \item \texttt{clustering/} – K-Means, Gath-Geva (with GPU acceleration).
        \item \texttt{optimization/} – Particle Swarm Optimization (PSO).
        \item \texttt{neural/} – Fuzzy Backpropagation Neural Network (FBPNN).
    \end{itemize}
    
    \item \texttt{metrics/} – Calculation of compression ratio, MSE, PSNR, entropy, and timing.
    
    \item \texttt{visualization/} – Generation of all plots and figures.
    
    \item \texttt{utils/} – Common helpers (e.g., bit I/O, entropy computation).
\end{itemize}

No external database is required; all results are stored in memory and can be exported as CSV or JSON for further analysis. Security considerations are minimal as the system runs locally and processes only user-supplied image files.

\subsection{Frontend Design}

The system does not include a graphical user interface; it is designed as a command-line tool and a Jupyter notebook that can be executed interactively. The notebook provides markdown cells that explain each algorithm, code cells that run the compression and visualization, and output cells that display the results. This design is well-suited for research and educational purposes.


\subsection{Data Flow}

The typical flow of data utilized for any single algorithm is laid out precisely below.

\begin{enumerate}
    \item An image is actively loaded straight from the disk (e.g., \texttt{Lenna.png}). A NumPy array taking the explicit shape of $(H,W,C)$ is utilized to hold this raw data.
    
    \item Resizing the image down to a strictly fixed size (e.g., $256\times256$) is optionally performed. Keeping overall evaluation times highly reasonable is directly achieved by doing this.
    
    \item Pixel values are normalized heavily down to a tight $[0,1]$ scale. This step is only performed if it is explicitly required by the given algorithm (e.g., neural networks demand this heavily).
    
    \item The specific \texttt{compress()} method tied to the algorithm is directly invoked. A compressed representation (e.g., a bit string, a raw list of symbols, or a color palette joined by indices) is immediately returned. Necessary metadata (e.g., Huffman code tables or exact cluster centers) is also outputted.
    
    \item Reconstruction of the target image is achieved by invoking the \texttt{decompress()} method directly.
    
    \item Quality metrics are computed thoroughly. Compression ratio, MSE, PSNR, and raw execution time are evaluated by directly comparing the original arrays against the newly reconstructed arrays.
    
    \item Visualizations are generated utilizing Matplotlib. Glaring error maps and histograms fall directly under this visualization umbrella.
    
    \item All generated figures and hard metrics are stored away safely. Later analysis and final reporting relies completely on this saved data pool.
\end{enumerate}

An inner loop spanning over multiple iterations is included strictly for iterative algorithms (GathGeva, PSO, FBPNN). Intermediate results are heavily recorded by this specific loop so that accurate convergence plots can be drawn up later.






\subsection{Algorithms and Techniques}

The following subsections summarize the key algorithms implemented. Detailed pseudocode and mathematical formulations are provided in the corresponding sections of the appendix.

\subsection{Entropy Coding}

\begin{itemize}
    \item \textbf{Huffman coding}: Builds a binary tree based on symbol frequencies and assigns variable-length codes. The codes are prefix-free and optimal for a given distribution when using integer-length codes.
    
    \item \textbf{Arithmetic coding}: Uses fractional bit assignments by representing the entire message as a single number in \([0,1)\). The implementation uses integer arithmetic to avoid floating-point precision issues.
    
    \item \textbf{LZW}: Maintains a dictionary of previously seen substrings; encodes each substring as a reference to the dictionary.
\end{itemize}

\subsection{Spatial Domain Methods}

\begin{itemize}
    \item \textbf{Run-length encoding (RLE)}: Encodes consecutive identical pixel values as a pair (value, count). Works best on images with large uniform areas.
    
    \item \textbf{Variable block size compression}: Divides the image into blocks of different sizes based on local variance; each block is encoded using RLE, base+offset, block matching, or raw transmission.
    
    \item \textbf{Lossy+residual}: First compresses the image lossily (e.g., using DCT with few coefficients), then transmits the residual (difference between original and lossy image) losslessly. The combination achieves lossless reconstruction with a high overall compression ratio.
\end{itemize}

\subsection{Predictive Coding}

\begin{itemize}
    \item \textbf{JPEG lossless prediction}: Uses one of seven predictors based on the left (\(a\)), above (\(b\)), and above-left (\(c\)) neighbors. The prediction error is entropy-coded with Huffman or arithmetic coding.
\end{itemize}

\subsection{Context-Based Methods}

\begin{itemize}
    \item \textbf{LOCO-I (JPEG-LS)}: Employs the median edge detector (MED) for prediction and Golomb-Rice coding for residuals. Contexts are formed by quantizing local gradients.
    
    \item \textbf{CALIC}: Uses a gradient-adjusted predictor (7 cases) followed by context modeling of the prediction error. The context is defined by the quantized error energy \(\Delta = d_h + d_v + 2|e_w|\) and an 8-bit texture pattern \(B\). Error feedback refines the prediction.
\end{itemize}

\subsection{Transform Domain Methods}

\begin{itemize}
    \item \textbf{S-Transform}: A reversible integer-to-integer Haar transform:
    \[
    l[n] = \left\lfloor\frac{c[2n] + c[2n+1]}{2}\right\rfloor, \qquad h[n] = c[2n] - c[2n+1]
    \]
    
    \item \textbf{S+P Transform}: Extends the S-Transform by predicting the high-frequency coefficients from low-frequency differences.
    
    \item \textbf{Lifting wavelet transform}: Decomposes the image using the lifting scheme, which performs the transform in place and allows integer-to-integer mapping.
\end{itemize}

\subsection{Clustering-Based Compression}

\begin{itemize}
    \item \textbf{K-Means}: Partitions the RGB space into \(K\) clusters; each pixel is replaced by the index of its nearest cluster center. The palette (centers) and index map are stored.
    
    \item \textbf{Gath-Geva fuzzy clustering}: A fuzzy variant that models clusters as ellipsoids. The membership degree \(\mu_{ik}\) of pixel \(k\) to cluster \(i\) is used to weight the color palette. GPU acceleration (JAX) reduces execution time by a factor of 10–20.
\end{itemize}

\subsection{Optimization Methods}

\begin{itemize}
    \item \textbf{Particle Swarm Optimization (PSO)}: Treats a color palette as a particle; the swarm evolves to minimize the MSE between the original and compressed image. The velocity and position update rules are:
    \[
    v_i(t+1) = w v_i(t) + c_1 r_1 (p_i - x_i) + c_2 r_2 (g - x_i)
    \]
    \[
    x_i(t+1) = x_i(t) + v_i(t+1)
    \]
    where \(w\) is the inertia weight, \(c_1, c_2\) are acceleration constants, and \(r_1, r_2\) are random numbers in \([0,1]\).
\end{itemize}

\subsection{Neural Network Approach}

\begin{itemize}
    \item \textbf{Fuzzy Backpropagation Neural Network (FBPNN)}: Combines Gath-Geva fuzzy memberships with backpropagation. The weight update rule includes a fuzzy factor \(z_i = (\mu_{ik})^m\):
    \[
    \Delta w_{ij}^L = z_i \cdot \eta \cdot \delta_{pj}^{L-1} \cdot \text{out}_{pj}^L
    \]
    The network has 3 input nodes (RGB), 8 hidden nodes, and 15 output nodes (one per cluster).
\end{itemize}

    \item \textbf{Testing:} Pytest (unit tests for individual algorithms)
    
    \item \textbf{Documentation:} LaTeX (thesis), Markdown (code comments)
\end{itemize}

All experiments were run on an Apple M4 MacBook Pro with 16 GB of unified memory, using the Metal backend for GPU acceleration where applicable.
