\subsection{Evaluation Methodology}

All algorithms were evaluated using the following metrics:

\begin{itemize}
    \item \textbf{Compression Ratio:} \( R = \frac{\text{Original Size}}{\text{Compressed Size}} \)
    \item \textbf{Mean Squared Error (MSE):} \( \frac{1}{N}\sum (I_{ij} - \hat{I}_{ij})^2 \)
    \item \textbf{Peak Signal-to-Noise Ratio (PSNR):} \( 20\log_{10}(255/\sqrt{\text{MSE}}) \)
    \item \textbf{Execution Time:} Time taken for compression and decompression
\end{itemize}

\subsection{Experimental Setup}

\begin{itemize}
    \item \textbf{Hardware:} Apple M4 MacBook Pro with 16 GB unified memory
    \item \textbf{Test Images:} Lenna (512×512), Cameraman (512×512)
    \item \textbf{Image Format:} 8-bit grayscale and 24-bit RGB
    \item \textbf{Number of Tests:} Each algorithm was run 5 times; average results reported
\end{itemize}

\subsection{Performance Analysis}

Context-based algorithms (CALIC, LOCO-I) achieved the best compression ratios (1.45–1.52:1) due to their adaptive prediction and context modeling. Entropy coders (Huffman, Arithmetic) were fastest (0.1–0.3 seconds) but provided lower ratios (1.07–1.08:1). GPU-accelerated clustering algorithms (Gath-Geva, PSO, FBPNN) achieved speeds of 0.5–1.0 seconds per image on the Apple M4.
