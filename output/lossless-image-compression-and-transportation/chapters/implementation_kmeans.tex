\subsubsection{K‑Means Clustering for Image Compression}

\paragraph{Introduction}
K‑Means is a classic unsupervised machine learning algorithm used for clustering and vector quantisation. In the context of image compression, it reduces the number of distinct colours in an image by grouping similar pixel colours into $K$ clusters and replacing each pixel with the index of its nearest cluster centre (the palette). This technique, known as colour quantisation, can achieve substantial compression while preserving perceptual quality.

\paragraph{Mathematical Formulation}
Given a set of data points $\{x_1, x_2, \dots, x_N\}$ (pixel colours in RGB space) and a desired number of clusters $K$, K‑Means minimises the within‑cluster sum of squares:

\[
J(V) = \sum_{i=1}^{K} \sum_{j=1}^{c_i} \| x_{ij} - v_i \|^2,
\]

where:
\begin{itemize}
    \item $v_i$ is the centre (mean) of cluster $i$,
    \item $c_i$ is the number of points in cluster $i$,
    \item $\|x - v\|$ is the Euclidean distance.
\end{itemize}

The cluster centres are updated as the mean of all points assigned to that cluster:

\[
v_i = \frac{1}{c_i} \sum_{j=1}^{c_i} x_{ij}.
\]

The algorithm iteratively alternates between assignment (nearest centre) and update (recompute means) until convergence.

\paragraph{Algorithm Steps}
\begin{enumerate}
    \item Randomly initialise $K$ cluster centres.
    \item Assign each data point to the nearest centre (Euclidean distance).
    \item Recompute each centre as the mean of its assigned points.
    \item Repeat steps 2–3 until no point changes cluster (or a maximum number of iterations is reached).
\end{enumerate}

\paragraph{Image Compression with K‑Means}
Two standard test images were used: \textit{Lenna} ($256\times256$, RGB, originally 47,420 unique colours) and \textit{Cameraman} ($256\times256$, RGB, originally 256 unique colours). For each image, K‑Means was applied with $K = 2,4,8,16,32$. The compressed image is stored as a colour palette ($K$ RGB triples) plus an index map ($\log_2 K$ bits per pixel). The compression ratio is:

\[
\text{Ratio} = \frac{\text{Original bits}}{\text{Palette bits} + \text{Index bits}}.
\]

\paragraph{Discussion}
\begin{itemize}
    \item \textbf{Compression efficiency:} K‑Means colour quantisation is highly effective for images with limited colour palettes. For Cameraman, even $K=2$ achieves a ratio of $24:1$, though with visible posterisation.
    \item \textbf{Convergence speed:} The number of iterations required is moderate (7–81) and increases with $K$ and image complexity.
    \item \textbf{Objective function:} $J(V)$ decreases monotonically as $K$ increases, as expected.
    \item \textbf{Limitations:} K‑Means assumes spherical clusters and is sensitive to initialisation; however, for colour quantisation these limitations are rarely problematic.
\end{itemize}

\paragraph{Summary}
K‑Means clustering provides a simple yet powerful method for lossy image compression via colour quantisation. The algorithm minimises the within‑cluster sum of squares, iteratively refining cluster centres. Experimental results on standard test images demonstrate compression ratios ranging from $4.8:1$ to $24:1$, depending on the number of clusters. The trade‑off between compression ratio and image quality can be controlled by selecting $K$. The method is particularly suitable for applications where a limited colour palette is acceptable, such as in graphics, thumbnails, and legacy display systems.
